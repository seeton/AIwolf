# お題仕様

## 目的

1 ラウンドの会話を「ある共通お題」に縛ることで、文脈を狭め、見抜き合いの判断材料を「お題への乗り方」に集中させる。

## カードの構造 (元 PR `web/app.js` の `TOPICS` から)

- `title` — 表示用のお題名
- `description` — 導入文 (UI 表示用)
- `keywords` — オフトピック判定用キーワード集合
- `prompts` — お題に乗りやすい書き出しの例
- `details` — 会話を深めるためのフレーバー語彙

## オフトピック判定

- 単純なキーワードマッチ (`isOnTopic` 相当)
- いずれかのキーワードを含めば通す、含まなければ弾く
- 弾いた場合は UI に「お題外の発言は禁止です」を表示し再入力させる

### 限界

- キーワードを混入させれば実質オフトピックでも通る
- 将来的に LLM 判定 / 埋め込みベースの類似度判定に置き換え可

## 未決事項

- カード追加運用 (誰が、レビューフローはどうするか)
- 難易度ラベル (議論しやすさ / 人格バレしやすさ)
- 同じお題が短期に再出題されない仕組み
- お題ごとの推奨ラウンド数
